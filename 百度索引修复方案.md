# 百度索引问题修复方案

## 背景

网站 https://www.apiuspro.cn 已通过百度站长验证并提交 URL，但一直没有索引记录和点击量。

## 诊断结果

经代码审查，发现以下问题：

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 3 个重要页面是纯 CSR | 严重 | `/tutorial`、`/app/[id]`、`/cloud-api` 使用 `'use client'`，百度爬虫看到空 HTML |
| `/learn` 页面未入 sitemap | 严重 | 最近新增的 learn 系列页面不在 sitemap 中 |
| `/app/[id]` 无 metadata | 高 | 没有 `generateMetadata`，百度看到空标题 |
| `/tutorial` 无 metadata | 高 | 同上 |
| `/cloud-api` 无 metadata | 高 | 同上 |
| 服务器可能在海外 | 高 | Coze 部署，百度爬虫可能访问困难 |

## 修复步骤

### 第一步：验证服务器可达性（手动操作）

在百度站长平台操作：
1. 登录 https://ziyuan.baidu.com
2. 进入你的网站 → 网站管理
3. 左侧菜单 → 抓取诊断
4. 输入 `https://www.apiuspro.cn` → 测试抓取
5. 记录结果：如果显示「抓取成功」且能看到完整 HTML，说明服务器可达；如果超时或失败，需要先解决服务器可达性

### 第二步：Sitemap 添加 learn 页面（最小改动）

**涉及文件**：`src/app/sitemap.ts`

### 第三步：修复 CSR 页面为 SSR + metadata（核心修复）

**涉及文件**：
- `src/app/app/[id]/page.tsx` — 拆分为 server layout + client 内容
- `src/app/cloud-api/page.tsx` — 改为 server component + client 子组件
- `src/app/tutorial/page.tsx` — 改为 server component + client 子组件

**原则**：
- 页面组件（page.tsx）改为 server component
- 交互逻辑（搜索、筛选、tab 切换）拆到 client 子组件
- 每个页面导出 `generateMetadata` 或静态 `metadata`

### 第四步：百度站长主动推送

修复部署后：
1. 在百度站长平台重新抓取诊断
2. 手动推送核心页面 URL
3. 提交更新后的 sitemap

### 提示词 2：修复 /app/[id] 页面为 SSR

```
请将 src/app/app/[id]/page.tsx 从纯 CSR 改为 SSR + CSR 混合模式。

当前问题：
- 页面以 'use client' 开头，百度爬虫看不到任何内容
- 没有 generateMetadata，百度看到空标题和描述

修复方案：
1. 将 page.tsx 改为 server component（去掉 'use client'）
2. 导出 generateMetadata 函数，从 appTutorials 中根据 params.id 获取数据，生成 title/description
3. 将所有交互逻辑（useState、useEffect、IntersectionObserver、event handlers）提取到一个新的 client 子组件 AppTutorialContent.tsx
4. page.tsx 作为 server component 负责：
   - 调用 generateMetadata 返回 SEO 信息
   - 渲染结构化数据（ArticleSchema、BreadcrumbSchema、HowToSchema）
   - 传入数据给 client 子组件
5. client 子组件接收 data props，负责交互逻辑

TDK 可以从 src/lib/tdk.ts 的 getTdkConfig 函数获取，参考教程页的写法：
```
import { generateMetadata as generateTdkMetadata } from '@/lib/tdk';
export function generateMetadata({ params }) {
  return params.then(({ id }) => generateTdkMetadata('/app/:id', { id, name: ... }));
}
```

先读取现有 page.tsx 和 src/lib/tdk.ts，理解现有结构后再修改。不要删除任何功能。
```

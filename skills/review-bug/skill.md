# Bug 审查 Skill

审查项目代码中的潜在 bug、安全漏洞、性能问题和代码质量问题。

## 使用方式

```
/review-bug                  # 审查整个项目
/review-bug src/app          # 审查指定目录
/review-bug src/app/page.tsx # 审查指定文件
```

## 审查维度（按优先级）

### 1. 功能 Bug
- 交互逻辑缺陷（按钮禁用条件、表单验证、事件处理）
- 状态管理问题（竞态条件、useEffect 清理、闭包陷阱）
- 数据渲染问题（列表 key、条件渲染遗漏、空值处理）
- 路由/导航问题（404、死链、参数解析）

### 2. 响应式布局 & 移动端适配
- 360px 宽度下内容溢出（固定宽度、min-w-、mx-8 等）
- flex/grid 布局在小屏的表现（gap-8、w-52 等固定尺寸）
- 表格/代码块横向溢出（缺少 overflow-x-auto）
- 触摸目标尺寸（按钮/链接 < 44px）
- 文字截断（truncate 用在不该截断的地方）

### 3. 深色模式
- 颜色类缺少 `dark:` 变体（bg-emerald-50、text-sky-800 等）
- 硬编码颜色值未适配深色背景
- 代码块/语法高亮深色模式不可读
- 对比度不足（text-muted-foreground 在深色模式太灰）

### 4. 无障碍 (a11y)
- 缺少 alt 属性（图片）
- 缺少 aria 标签（交互元素）
- 键盘导航问题（焦点陷阱、Tab 顺序）
- 屏幕阅读器兼容（role 属性冲突、嵌套交互元素）

### 5. 性能
- 第三方脚本阻塞渲染（beforeInteractive 策略）
- 大图片未优化（缺少 sizes、未用 next/image）
- 不必要的重渲染（缺少依赖数组、内联对象/函数）
- Bundle 体积问题（动态导入缺失）

### 6. 代码质量
- TypeScript 类型断言滥用（as any、as Type 强制断言）
- 防御性编码缺失（未检查 null/undefined）
- 死代码/未使用导入
- 编码问题（文件编码损坏、乱码）

### 7. SEO & 元数据
- meta 标签缺失或错误
- 结构化数据问题
- Open Graph / Twitter Card 配置

## 输出格式

### 汇总报告

```
## Bug 审查报告

**审查范围**: [目录/文件列表]
**审查时间**: YYYY-MM-DD HH:MM
**发现总数**: X 个问题

### 严重程度分布
- Critical: X 个
- High: X 个
- Medium: X 个
- Low: X 个
```

### 单个 Bug 格式

```
### Bug #N（严重程度）

**文件**: `path/to/file.tsx`，行 XX-XX
**问题**: 简洁描述
**影响**: 用户/系统影响
**修复**: 具体修复方案（包含代码示例）
```

严重程度定义：
- **Critical**: 数据丢失、安全漏洞、页面崩溃、编码损坏
- **High**: 核心功能失效、深色模式完全不可用、移动端布局严重错乱
- **Medium**: 样式问题、交互体验降级、性能影响
- **Low**: 代码风格、不一致、小优化

## 审查流程

1. **扫描阶段**: 遍历目标文件，按维度逐项检查
2. **验证阶段**: 对可疑代码确认是否为实际 bug（非设计偏好）
3. **分级阶段**: 按严重程度排序
4. **报告阶段**: 输出结构化报告，包含修复方案

## 注意事项

- 只报告实际 bug，不报告设计偏好或风格问题
- 每个 bug 必须包含文件路径、行号、问题描述、影响、修复方案
- 修复方案应具体到代码级别，不要只说"建议优化"
- 对于深色模式问题，给出完整的 dark: 变体建议
- 对于响应式问题，给出具体的断点和类名建议

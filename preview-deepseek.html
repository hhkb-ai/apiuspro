接入步骤

以下是接入 DeepSeek API 的完整流程，从注册到首次调用。每一步都标注了操作位置和成功标志，新手按顺序操作即可。

1

注册并完成实名认证

先进入 DeepSeek 开放平台（platform.deepseek.com），用手机号或邮箱注册账号。完成注册后，在账号设置中进行实名认证——个人开发者提交身份证信息，企业用户提交营业执照。认证通过后才能创建 API Key 和充值。

2

创建并保存 API Key

登录 DeepSeek 控制台，进入 API Keys 管理页面，点击 "Create API Key" 创建密钥。创建后立即复制保存——API Key 通常只完整显示一次。

不要把 API Key 写入前端代码、公开仓库、截图或聊天记录中。建议存储在 .env 文件或环境变量管理工具中。
3

充值并确认余额

进入账单或 Usage 页面，选择合适的充值档位（10、20、50 或 100 元），使用支付宝或微信完成支付。支付成功后确认账户余额大于 0。

4

选择模型并跑通测试

DeepSeek 推荐优先使用 deepseek-v4-flash（实时对话、高频调用）或 deepseek-v4-pro（复杂推理、长文档）。旧别名 deepseek-chat 和 deepseek-reasoner 将在 2026 年 7 月退役，新项目不要继续使用。

python 
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com",
)

response = client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=[{"role": "user", "content": "你好，请用一句话介绍你自己。"}],
)

print(response.choices[0].message.content)
使用 OpenAI SDK 调用 DeepSeek API。只需将 base_url 指向 https://api.deepseek.com，api_key 从环境变量读取。成功执行并打印回复即代表配置正确。
5

排查常见问题

如果调用失败，优先检查：API Key 是否复制完整、Authorization 格式是否正确、Base URL 是否为 https://api.deepseek.com、模型名称是否拼写正确。

代码示例

DeepSeek 兼容 OpenAI 接口格式，以下示例使用 OpenAI SDK 调用。只需修改 base_url 和 api_key。

Python (OpenAI SDK)

python 
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com",
)

response = client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=[{"role": "user", "content": "你好，请用一句话介绍你自己。"}],
)

print(response.choices[0].message.content)
使用 OpenAI SDK 调用 DeepSeek API。将 base_url 指向 https://api.deepseek.com，api_key 从环境变量读取。

cURL

bash 
curl -X POST https://api.deepseek.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $DEEPSEEK_API_KEY" \
  -d '{
    "model": "deepseek-v4-flash",
    "messages": [
      { "role": "user", "content": "你好，请介绍一下自己" }
    ],
    "temperature": 0.7
  }'

关键参数说明

  • base_url：必须指向 https://api.deepseek.com，不是 OpenAI 的地址
  • model：推荐使用 deepseek-v4-flash（推荐，1M 上下文）
  • temperature：控制输出随机性，0.7 适合大多数场景

DeepSeek API 国内可以访问吗？

DeepSeek API 更适合国内用户优先尝试，通常可以直接访问。正式接入前仍建议用免费额度或小额充值跑通真实任务。

DeepSeek API Key 在哪里创建？

进入 DeepSeek 控制台后，在 API Key、密钥管理或开发者设置页面创建密钥。可以继续查看本站的 DeepSeek 购买教程。

DeepSeek API Key 泄露了怎么办？

立即到控制台删除泄露的 Key 并重新生成。建议把 Key 存在环境变量或 .env 文件中，不要提交到 Git 仓库。

DeepSeek API 报错 401 / Invalid API Key？

首先确认 API Key 是否已复制完整，再检查 Authorization 请求头格式是否为 "Bearer sk-xxx"。如果 Key 已删除或过期，需要到控制台重新创建。

DeepSeek API 适合什么场景？

DeepSeek 适合的方向包括：高性价比、代码能力强、推理模型、开源。如果你还不确定具体选择，可以先查看场景推荐页。

至少确认官网入口、免费额度或计费规则、Base URL、模型名、API Key 保存方式和限速规则。不要把 API Key 写入前端代码。

# 全站优化方案：性能 + SEO + 移动端交互

> 基于代码审查，列出所有可执行的具体修改项。每项标注涉及文件、改动内容和预期效果。你看了之后告诉我做哪些。

---

## 一、性能优化（影响搜索排名的 Core Web Vitals）

### P1-1 首页消除双组件渲染

**问题：** `home-client.tsx` 第 493-499 行同时渲染两套完整 HomeShell：

```tsx
<div className="md:hidden"><HomeShell variant="mobile" /></div>
<div className="hidden md:block"><HomeShell variant="desktop" /></div>
```

移动端用户下载了 desktop 版本的全部 JS bundle（包括 useState、useRouter、SearchBar、Hero、PurchaseTutorials 等所有组件代码），但通过 CSS `hidden` 隐藏。这导致：
- JS 解析时间翻倍，拖慢移动端 LCP 和 INP
- hydration 时 React 需要为两个不可见/可见的组件树做 diff

**改法：** 合并为一个 HomeShell 组件，内部用 `useMediaQuery` 或 Tailwind 断点类控制布局差异。现有代码中 mobile 和 desktop 的差异仅在于：
- Hero 标题字号（`text-[28px]` vs `text-4xl`）
- Hero 描述文字长度
- 新手路线面板 compact 模式
- 搜索结果数量（6 vs 8）

这些完全可以用一个组件 + 响应式类解决，不需要维护两套。

**涉及文件：** `src/app/home-client.tsx`

**预期效果：** 移动端 JS bundle 减少约 30-40%，LCP 和 INP 指标改善。

---

### P1-2 教程详情页图片 lazy loading

**问题：** 教程步骤中的图片（`step.image`）使用 `next/image` 但没有显式设置 `loading="lazy"`。虽然 Next.js 默认对非首屏图片 lazy load，但教程页首屏可能有 2-3 张图同时加载。

**改法：** 给首屏以下的图片加 `loading="lazy"`，首屏第一张图保持 `priority`。

**涉及文件：** `src/app/tutorial/[id]/page.tsx`

---

### P1-3 分析脚本加载优化

**问题：** `layout.tsx` 中百度统计、51.la SDK 使用 `beforeInteractive` 和 `afterInteractive` 策略，这些脚本在页面交互前就开始加载，占用主线程时间。

**改法：**
- 百度统计 `LA_COLLECT` 从 `beforeInteractive` 改为 `lazyOnload`（页面完全空闲后再加载）
- 51.la init 脚本也改为 `lazyOnload`
- 百度自动推送保持 `afterInteractive`（需要尽早执行）

**涉及文件：** `src/app/layout.tsx`

**预期效果：** 减少主线程阻塞时间，改善 INP。

## 二、SEO 优化（提升搜索排名和点击率）

### S1-1 教程页补充 FAQ 结构化数据

**问题：** 只有 `api/[id]` 详情页有 FAQSchema（`getApiFaqItems`），但教程页 `/tutorial/[id]` 没有。教程页是用户搜索"XX怎么买"、"XX API Key 怎么创建"时的主要落地页，缺少 FAQ 结构化数据意味着 Google 不会在搜索结果中展示 FAQ 富文本片段。

**改法：** 在 `tutorial/[id]/page.tsx` 中为每个教程添加 3-4 个常见问题的 FAQSchema，内容可以从现有步骤中提炼：
- "XX API 国内能直接用吗？"
- "XX API Key 在哪里创建？"
- "XX 的价格是多少？"
- "第一次调用 XX API 需要什么？"

**涉及文件：** `src/app/tutorial/[id]/page.tsx`

**预期效果：** 教程页在搜索结果中展示 FAQ 富文本片段，CTR 预期提升 15-25%。

---

### S1-2 场景推荐页补充 FAQ 结构化数据

**问题：** `/use-case` 和 `/use-case/[id]` 页面没有 FAQSchema。用户搜索"编程用哪个AI API"、"做知识库用什么模型"时，这些页面是落地页。

**改法：** 每个场景页添加 2-3 个 FAQ，如"编程开发推荐哪些 AI API？"、"免费的编程 AI 有哪些？"

**涉及文件：** `src/app/use-case/[id]/page.tsx`

---

### S1-3 购买教程列表页补充结构化数据

**问题：** `/tutorial` 列表页有 `BreadcrumbSchema` 但没有 `ItemListSchema`。首页已经用了 `ItemListSchema` 列出 API 和应用教程，但购买教程列表页本身缺少。

**改法：** 在 `tutorial/page.tsx` 中添加 `ItemListSchema`，列出有教程的 API 项。

**涉及文件：** `src/app/tutorial/page.tsx`

---

### S1-4 优化首页 title 标签的搜索意图匹配

**问题：** 当前首页 title 是 "API知识站 - AI API 选型、购买、接入一站式指南"。用户搜索时更可能用的词是"AI API 怎么买"、"API 购买教程"、"DeepSeek 怎么用"。title 中的"一站式指南"偏官方宣传，不够匹配用户搜索意图。

**改法：** 调整为更贴近用户搜索词的表述，例如 "AI API 购买教程、价格对比与接入指南 | API知识站"。description 保持现有内容不变。

**涉及文件：** `src/lib/tdk.json`（首页 `/` 条目）

**预期效果：** 提升首页在"AI API 购买"类搜索词下的 CTR。

---

### S1-5 教程页 title 加入年份标签

**问题：** 教程类内容加年份（如"2026"）能显著提升 CTR，因为用户会认为内容是最新的。当前 `/local-deploy` 已经加了 "2026"，但 `/tutorial` 和 `/tutorial/:id` 没有。

**改法：**
- `/tutorial` title 改为 "AI API 购买教程大全 2026 — ..."
- `/tutorial/:id` title 保持不变（每个 API 的价格变化频繁，固定年份可能很快过时）

**涉及文件：** `src/lib/tdk.json`

---

### S1-6 百度搜索主动推送覆盖更多页面

**问题：** 当前百度自动推送脚本只推送当前页面。但百度站长平台建议推送 sitemap 中的所有重要页面。

**改法：** 在百度推送脚本中批量提交 sitemap 中的 URL，而非仅推送当前页。或者确保 sitemap.xml 已包含所有重要页面并被百度收录。

**涉及文件：** 检查 `src/app/sitemap.ts` 是否覆盖所有页面（教程详情、API 详情、场景推荐详情、应用教程详情、测评详情）。

## 三、移动端交互优化（提升用户体验和停留时间）

### M1-1 全局触摸反馈

**问题：** 所有可交互元素（Link、Button、卡片）在移动端缺少按压反馈（active 样式）。用户点击后没有视觉确认，交互感觉"虚"。

**改法：** 在 `globals.css` 中添加全局 active 样式：

```css
@media (hover: none) {
  a, button, [role="button"] {
    -webkit-tap-highlight-color: transparent;
  }
}
```

然后在关键组件上添加 `active:scale-[0.98] active:transition-transform active:duration-100`。

**涉及文件：** `src/app/globals.css`、`src/app/home-client.tsx`、`src/components/content/CloudApiContent.tsx`、`src/components/content/TutorialListContent.tsx`

---

### M1-2 菜单动画

**问题：** SidebarLayout 的移动端菜单（第 104-131 行）没有过渡动画，瞬间弹出/消失。

**改法：** globals.css 中已有 `mobile-sheet-fade` 和 `mobile-sheet-panel` 动画定义，但 SidebarLayout 没有使用。将菜单遮罩加上 `mobile-sheet-fade` 类，导航列表加上 `mobile-sheet-panel` 类。

**涉及文件：** `src/components/layout/SidebarLayout.tsx`

---

### M1-3 教程详情页 header 移动端精简

**问题：** 教程页 header 在小屏手机上内容密度过高（返回导航 + 分隔符 + 品牌图标 + 标题 + 代理标签 + 访问官网按钮），导致文字截断或布局挤压。

**改法：** 移动端（`< md`）只保留：
- 返回按钮（左）
- 标题文字截断（中）
- "官网" 按钮（右）
- 代理标签移到标题下方或省略

**涉及文件：** `src/app/tutorial/[id]/page.tsx`

---

### M1-4 搜索框添加清除按钮

**问题：** 搜索框没有清除按钮，用户只能逐字退格或全选删除。

**改法：** 在 `SearchBar` 组件中，当 `query` 非空时显示一个 × 按钮，点击清空输入并保持焦点。

**涉及文件：** `src/app/home-client.tsx`（SearchBar 组件）

---

### M1-5 首页热门教程改为横向滚动

**问题：** 首页 PurchaseTutorials 的 4 张卡片在移动端是单列排列（`grid-cols-1`），占屏空间大。

**改法：** 移动端改为横向滚动容器（`flex overflow-x-auto snap-x snap-mandatory`），每张卡 `min-w-[80vw] snap-start`，桌面端保持 grid。

**涉及文件：** `src/app/home-client.tsx`（PurchaseTutorials 组件）

---

### M1-6 首页内容节奏调整

**问题：** 移动端 Hero + 新手路线 + 热门教程 + 用户入口卡片全部纵向堆叠，首屏信息过载。

**改法：**
- Hero 区域移动端 `py-8` 缩为 `py-6`，`space-y-6` 缩为 `space-y-4`
- 隐藏移动端的 eyebrow 文字 "AI 与 API 学习导航"
- 用户入口卡片 `min-h-[220px]` 改为 `min-h-[160px]`

**涉及文件：** `src/app/home-client.tsx`

---

### M1-7 API 列表页筛选栏 sticky

**问题：** CloudApiContent 的搜索框和筛选 tabs 在滚动后消失，用户丢失当前位置。

**改法：** 将搜索 + tabs 容器改为 `sticky top-12 z-40 bg-background/95 backdrop-blur`。

**涉及文件：** `src/components/content/CloudApiContent.tsx`

---

### M1-8 内容页 padding 统一

**问题：** 多个内容页使用 `p-6 lg:p-8`，在手机上水平 padding 24px 偏大（390px 屏幕 - 48px padding = 342px 内容宽度）。

**改法：** 移动端 padding 改为 `px-4 py-6 lg:p-8`，水平 padding 16px，内容宽度增加 16px。

**涉及文件：** `src/app/api/[id]/page.tsx`、`src/app/tutorial/[id]/page.tsx`

# API知识站主题拆解

## 1. 主题

**主题名称**：API知识站 — AI API 选型、购买、接入一站式指南

**核心主题**：
- 帮助用户快速找到可用的 AI API。
- 解决“去哪买、怎么买、怎么拿 Key、怎么接入、怎么选择模型”的连续问题。
- 将分散的官网入口、购买教程、API 测评、工具接入、本地部署和场景推荐整合成一条清晰路径。

**主题关键词**：
- AI API 怎么选
- AI API 怎么买
- API Key 获取教程
- 国内可用 AI API
- 无需代理 AI API
- AI API 价格对比
- Claude Code 配置 API
- Ollama 本地部署
- AI 编程助手 API 推荐

## 2. 定位

**站点定位**：面向初学者、个人开发者、AI 工具用户和小团队的 AI API 学习与选型网站。

**用户群体**：
- 刚开始接触 AI API 的新手。
- 想获取 DeepSeek、通义千问、OpenAI、Claude、Gemini 等 API Key 的用户。
- 需要给 Claude Code、Codex、OpenCode、OpenClaw 等工具配置 API 的开发者。
- 想比较国内直连、免费额度、价格、模型能力的个人开发者。
- 想用本地部署方案降低成本或保护隐私的用户。

**核心用户问题**：
- 哪些 AI API 国内能直接用？
- API Key 在哪里创建？
- 哪个 API 有免费额度？
- DeepSeek、通义千问、Claude、OpenAI 该怎么选？
- Claude Code / Codex 怎么接入国产 API？
- 本地模型和云端 API 哪个更适合我？

**内容定位**：
- 不做泛泛的 AI 新闻站。
- 不做单纯官网导航。
- 重点做“可执行指南”：入口、步骤、检查清单、适用场景、排错建议。

## 3. 亮点

**内容亮点**：
- 覆盖从 API 选择到接入使用的完整链路。
- 每个 API 详情页强调官网入口、访问条件、免费额度、购买教程、测评入口。
- 购买教程按步骤组织，适合新手照着操作。
- 场景推荐按真实任务拆分：编程、知识库、内容创作、客服、翻译、教育。
- 本地部署页解释 Ollama、Qwen、Gemma、本地模型和云端 API 的取舍。

**体验亮点**：
- 首页直接提供搜索入口，支持 API、教程、应用和页面搜索。
- 搜索已支持模糊匹配、别名、错拼和中英文混合查询。
- API 列表支持按“无需代理 / 需要代理”筛选。
- 详情页有返回列表位置的体验设计，适合连续浏览多个 API。
- 移动端侧边导航可展开、关闭，并支持 Esc 退出。

**SEO 亮点**：
- 首页、API 详情、教程详情、测评详情、场景页均配置 metadata。
- 使用长尾关键词池覆盖“怎么买、怎么选、免费额度、国内直连、API Key、安全保存”等搜索意图。
- robots 允许公开页面被抓取，sitemap 覆盖主要静态页和动态详情页。
- 结构化数据覆盖 Website、Organization、Breadcrumb、FAQ、HowTo、Article、ItemList。

## 4. 风格

**整体风格**：
- 实用、直接、偏工具站。
- 面向新手，但不使用过度营销语气。
- 信息密度高，优先给用户下一步行动。

**视觉风格**：
- 基于 shadcn/ui 与 Tailwind 的简洁卡片式结构。
- 以边框、浅色卡片、徽标和小标题组织信息。
- 首页偏“搜索 + 快速入口 + 精选内容”。
- 列表页偏“筛选 + 卡片 + 检查清单”。
- 详情页偏“步骤化教程 + 注意事项 + 结构化导航”。

**文案风格**：
- 标题明确：直接写用户正在搜索的问题。
- 描述具体：包含官网入口、免费额度、Base URL、API Key、限速、计费等实际信息。
- 少用抽象形容词，多用可验证条件。
- 避免承诺“保证可用”，强调“以官方控制台为准”和“先小额验证”。

## 5. 强调

**需要反复强调的内容**：
- 国内用户优先看无需代理 API。
- 购买前先确认免费额度、充值门槛、限速和计费单位。
- API Key 只显示一次，必须安全保存。
- 不要把 API Key 写到前端代码、截图或公开仓库。
- 正式上线前要用真实任务测试质量、延迟、成本和失败率。
- 选择模型不要只看热度，要按实际场景验证。

**产品强调点**：
- “更快找到可用的 AI API”
- “少走接入弯路”
- “从搜索问题到可执行步骤”
- “先小额测试，再正式迁移”
- “按国内直连、免费额度、模型能力快速筛选”

**SEO 强调点**：
- 长尾关键词应自然落在标题、副标题、说明段、检查清单和 FAQ 中。
- 每个页面只围绕一个主要搜索意图展开。
- API 详情页强调品牌词 + “API 怎么买 / API Key / 免费额度 / 国内能不能用”。
- 教程页强调“注册、认证、充值、创建 API Key、首次调用”。
- 场景页强调“写代码用哪个 AI / 知识库用哪个 API / 客服机器人用什么模型”。

## 6. 页面顺序

### 6.1 首页 `/`

**作用**：总入口，承接品牌词和高意图搜索。

**页面顺序**：
1. 顶部导航：API 官网、API 测评、购买教程、API 应用、本地部署。
2. 主视觉：说明站点价值和核心定位。
3. 搜索框：支持 API、教程、应用、页面模糊搜索。
4. 数据概览：API 官网入口、购买教程、应用教程数量。
5. 高意图入口：AI API 怎么选、API Key 怎么拿、AI 工具怎么接入、写代码用哪个 AI。
6. 新手快速入门：选择 API、注册购买、接入使用。
7. 内容入口：API 列表、购买教程、本地部署。
8. 常用 API：DeepSeek、OpenAI、通义千问、Claude、Gemini、智谱 GLM、Kimi。
9. 新手决策指南：国内用户、免费额度、真实任务测试。
10. 精选 API：展示核心 API 卡片。
11. API 应用教程：展示工具接入教程。
12. 更新日志。

### 6.2 API 官网列表 `/cloud-api`

**作用**：帮助用户筛选 API 服务。

**页面顺序**：
1. 页面标题：API 官网链接。
2. 决策卡片：第一次接 API、写代码和调试项目、做知识库或文档处理。
3. 快速任务按钮：代码生成、多模态检索、长文本处理、数学推理、免费试用。
4. 搜索框：支持模糊搜索和别名搜索。
5. 筛选按钮：全部、无需代理、需要代理。
6. 无需代理 API 列表。
7. 需要代理 API 列表。
8. AI API 选择检查清单。

7. 场景推荐入口。

## 7. 后续内容扩展方向

**优先扩展方向**：
- 每个 API 增加“适合谁 / 不适合谁”。
- 每个教程增加“失败排查”小节。
- 每个场景增加真实测试样本。
- API 测评增加“新手结论”和“生产环境建议”。
- 本地部署增加不同硬件配置下的推荐模型。

**避免方向**：
- 避免写过泛的 AI 新闻。
- 避免堆砌关键词。
- 避免承诺长期价格或免费额度固定。
- 避免提供账号代注册、代充、代理绕过等高风险服务表达。
